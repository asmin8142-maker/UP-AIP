# Выводы

## Почему merge sort быстрее bubble sort?

Merge sort имеет сложность O(n log n), а bubble sort — O(n²).  
При увеличении размера входных данных bubble sort становится значительно медленнее.

## Как влияет тип входных данных?

Bubble sort:
- отсортированный массив — лучший случай;
- обратно отсортированный — худший случай.

Merge sort почти не зависит от порядка данных.

## Почему Fibonacci ограничен n ≤ 30?

Рекурсивный Fibonacci имеет экспоненциальную сложность O(2ⁿ), поэтому время выполнения очень быстро растёт.
